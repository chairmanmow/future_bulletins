# future_bulletins

FUTURELAND announcement carousel for Synchronet. Presents posts from the
`LOCAL-NOTICES` message base as full-screen slides: TDF-font headlines with a
shimmer sweep, dates in TDF where the terminal is tall enough, the message
body in a box whose border colors travel clockwise, scrolling TDF marquee
bands of the BBS name/location/user greeting in the background, and Clippy
(borrowed from future_signup) with a speech bubble in the bottom-left.

## Modes

- `?flbullet.js` — menu / external-program launch: shows every bulletin.
- `?flbullet.js logon` — logon sequence: shows only bulletins the user has
  not read; exits silently with no output when the user is caught up. Wired
  up in `/sbbs/mods/customlogon.js`.

Bulletins are ordered newest first. Read state is tracked per user and per
message in `data/flbullet_read.json` (`{ "<usernum>": { "<msgnum>":
<timestamp> } }`) — viewing a bulletin marks exactly that bulletin, and `M`
marks everything at once without reading. Marks for messages deleted from
the base are pruned on save.

Guests are always shown everything and never tracked. Non-ANSI terminals get
a plain sequential dump.

## Keys

- `←`/`→`, `P`/`N`, space, enter — previous / next bulletin (newest first)
- `↑`/`↓`, `PgUp`/`PgDn`, `Home`/`End` — scroll the message body
- `1`-`9` — jump to bulletin
- `M` — mark all bulletins as read
- `A` — post a new announcement to LOCAL-NOTICES (sysop only)
- `E` — edit the current announcement's title or body (sysop only). Title
  edits rewrite the header in place; body edits hand off to the sysop's
  configured message editor via `bbs.edit_msg()`, which replaces the body
  under the same message number — so per-user read state, carousel order,
  and the original date are all preserved.
- `D` — delete the current announcement (sysop only). This is Synchronet's
  standard soft delete (`MSG_DELETE` header attribute); the message
  disappears from the carousel immediately and is physically purged by
  normal message-base maintenance.
- `Q`/`ESC` — quit
- Mouse hotspots on the status bar; click Clippy for a treat

On terminals under 20 rows each bulletin becomes two screens: a TDF title
card, then the message body (any key dismisses the title).

## Files

- `flbullet.js` — the program
- `paperclip.sprite.json` / `paperclip_pablo.bin` — Clippy sprite (copied
  from `/sbbs/mods/future_signup/`)
- `fl_announce_banner.bin` — legacy header art (no longer used)

TDF fonts are read from `ctrl/tdfonts/`; the height-tier map used for
semi-random font selection is `data/figlet_font_map.json` (generated by the
webv4_custom tooling).

Many 90s TDF fonts ship with the artist's advertisement baked into glyph
data, usually on punctuation characters (`@`, `!`, `~`, `|`). At load time
each glyph is scanned for readable ad vocabulary ("font", "thedraw",
"call"...); contaminated glyphs are disabled (so the font is skipped for any
text needing them) and fonts that are mostly ad (`cfh-hoer`, `kevin4`,
`white-bl`) are dropped entirely.
